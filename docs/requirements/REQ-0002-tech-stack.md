# REQ-0002 - 客户端技术栈与主题/i18n 规范

状态：Draft  
创建日期：2026-02-05  
关联 BDD：`docs/bdd/REQ-0002-tech-stack.feature`

## 背景 / 问题

当前项目文档缺少客户端技术栈、主题与国际化能力的统一约定，导致后续实现与协作标准不一致。

## 目标

- 明确客户端技术栈：TypeScript + Electron + Vite。
- 明确 UI 与样式体系：shadcn/ui + Tailwind CSS。
- 明确国际化、主题模式与主题色可配置能力。
- 明确图标体系使用 Lucide。

## 非目标

- 不定义具体页面实现细节。
- 不规定业务逻辑与数据接口。

## 范围

### In Scope

- 技术栈约束与构建工具约定。
- i18n 机制与默认语言要求。
- 亮色/暗黑主题模式与用户可配置主题色。
- 图标库选型约定。

### Out of Scope

- 主题配色与 Token 的具体数值。
- Electron 主进程、预加载脚本的工程结构细节。

## 用户故事

- 作为产品负责人，我希望明确客户端技术栈与 UI 体系，以便团队统一实现标准。
- 作为用户，我希望客户端支持亮色/暗黑模式与自定义主题色，以便适配个人偏好。
- 作为全球用户，我希望客户端支持 i18n，以便使用本地语言。

## 验收标准（与 BDD 对齐）

- [ ] 项目文档明确技术栈与构建工具约定。
- [ ] 项目文档明确 i18n 与主题模式要求。
- [ ] 项目文档明确主题色可配置与 Lucide 图标库。

## 风险与合规（如适用）

- 需确保主题色配置不影响可访问性（对比度与可读性）。

## 开放问题

- i18n 默认语言优先级与语言包加载策略待补充。
