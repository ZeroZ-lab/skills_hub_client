# REQ-0003 - 客户端信息架构：左侧导航 + 基础页面骨架

状态：Draft  
创建日期：2026-02-05  
关联 BDD：`docs/bdd/REQ-0003-requirement.feature`

## 背景 / 问题

当前客户端仍是模板页，缺少与设计文档一致的“左侧导航 + 多页面信息架构”，导致后续功能（Skills/MCP/Projects/Profile/Market/Doctor/Logs）无法按页面承载与演进。

## 目标

- 建立与设计文档一致的信息架构：左侧导航 + 右侧主面板。
- 提供 MVP 页面骨架（可渲染、可切换、可扩展），支持后续逐步填充业务功能。
- 保留现有主题/主色/i18n 能力，并将其纳入新的 App Shell（顶部栏/设置入口）。

## 非目标

- 不实现 Skills/MCP/Projects 的真实安装/同步/接管/诊断逻辑（仅骨架与占位）。
- 不在本 REQ 中引入企业市场服务端或联网能力。

## 范围

### In Scope

- 左侧导航包含：Dashboard、Skills、MCP Servers、Projects、Profiles、Market、Doctor、Logs。
- 默认进入 Dashboard；点击导航项后切换到对应页面。
- 每个页面至少包含：页面标题、简要说明（占位），并预留列表/详情区域（可后续扩展）。
- 顶部栏展示：当前激活 Profile（占位）、主题切换、语言切换（沿用现有实现）。

### Out of Scope

- 页面内的复杂交互（diff 预览、审批流、审计查询、连接测试等）。
- 本地数据库（SQLite）落地（后续 REQ 单独推进；本阶段可用内存/静态数据占位）。

## 用户故事

- 作为用户，我希望在客户端中通过左侧导航快速切换到 Skills/MCP/Projects 等页面，以便按功能域管理能力资产。
- 作为开发者，我希望先有稳定的 App Shell 与页面骨架，以便按 REQ/BDD 逐步实现每个模块而不破坏整体结构。

## 验收标准（与 BDD 对齐）

- [ ] 应用启动后存在左侧导航，且包含设计文档中的全部导航项。
- [ ] 默认页面为 Dashboard；点击导航项可切换到对应页面并显示正确标题。
- [ ] 主题切换与语言切换在新 App Shell 中可用（行为与现有一致）。

## 风险与合规（如适用）

- Electron 渲染层路由/状态管理需保持简单可控，避免过早引入复杂依赖导致后续重构成本。

## 开放问题

- 后续是否引入路由库（如 React Router）？MVP 先采用轻量状态切换，后续视复杂度再定。
